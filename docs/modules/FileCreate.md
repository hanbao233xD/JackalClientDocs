FileCreate
文件创建
分类：File
描述：创建指定文件。

需求
- Unsafe：否
- Malicious：否
- 需要管理员权限：否
- 需要系统权限：否
- 需要 TrustedInstaller 权限：否
- 驱动依赖：否
- 是否 WIP：否
- 是否需要联网：否
- 版本属性：普通可用

介绍
FileCreate 用于按路径创建文件或文件夹。启用后会立即执行一次创建；若 `Keep` 开启，模块会在后台周期中持续检查并尝试补建。
模块支持直接创建目录，并可在创建后附加只读/隐藏/系统等属性。

配置项
- Path（路径）
类型：文本；默认：空；作用：指定待创建目标路径。建议：使用绝对路径并确认父目录存在。

- Keep（保持）
类型：布尔；默认：false；作用：开启后会周期性重建目标（目标不存在时）。建议：用于“守护文件/目录”场景，普通一次性创建请关闭。

- Notify（通知）
类型：布尔；默认：true；作用：控制是否提示创建成功/失败。建议：调试阶段开启，长期守护可按需关闭。

- UNC Path（UNC路径）
类型：布尔；默认：true；作用：自动把路径转换为 `\\?\` 长路径前缀，降低长路径导致的失败概率。建议：保持开启。

- Create Folder（创建文件夹）
类型：布尔；默认：false；作用：关闭时创建文件，开启时创建目录。建议：与目标类型保持一致，避免路径冲突。

- Readonly（只读）
类型：布尔；默认：false；作用：创建后附加只读属性。建议：仅在确有防修改需求时开启。

- Hidden（隐藏）
类型：布尔；默认：false；作用：创建后附加隐藏属性。建议：仅在管理策略明确时开启。

- System（系统）
类型：布尔；默认：false；作用：创建后附加系统属性。建议：谨慎开启，避免影响常规文件管理行为。

备注
若目标已存在，模块会直接返回并按通知策略提示，不会覆盖已有文件。
`Create Folder` 开启时，会先创建目录再设置属性；属性设置失败会单独报错。
